# [自进化 Skill：选股策略的自动迭代](https://inst.htsc.com/research/report?reportId=4501046409)：学习笔记

> 原文：华泰证券金工研报《自进化 Skill：选股策略的自动迭代》。\
> 本笔记以量化研究视角整理，重点关注：研究框架、模型设置、因子选择、优缺点，以及可迁移的
> alpha。\
> 仅作研究学习笔记，不构成投资建议。

------------------------------------------------------------------------

## 1. 一句话总结

这篇研报的核心思想是：

**利用 LLM Agent 将传统量化研究员"提出假设 → 修改策略 → 回测验证 →
继续优化"的人工流程自动化，通过 Skill
描述策略规则，并利用版本管理和样本隔离机制实现选股策略的自进化。**

文章的重点不是构造一个新的选股因子，而是探索：

> AI 是否可以成为自动化 Quant Researcher。

------------------------------------------------------------------------

## 2. 研究背景

传统量化研究流程：

1.  研究员提出投资逻辑；
2.  构造因子；
3.  编写策略代码；
4.  回测；
5.  分析结果；
6.  修改策略。

主要问题：

-   研究过程依赖个人经验；
-   大量试错过程无法沉淀；
-   参数调整缺乏系统记录；
-   策略优化容易陷入局部搜索。

本文借鉴 AutoResearch 思想，将研究流程转化为 Agent 可执行流程。

------------------------------------------------------------------------

## 3. AutoResearch 框架

整体框架：

    策略协议层
          ↓
    版本管理层
          ↓
    样本隔离层
          ↓
    Agent 自动迭代

------------------------------------------------------------------------

## 4. Skill：可执行研究协议

Skill 是本文最核心的概念。

它不是普通策略代码，而是一份结构化研究协议，包括：

-   投资逻辑；
-   股票池规则；
-   因子定义；
-   参数范围；
-   可修改模块；
-   评价指标；
-   回测约束。

其作用类似：

> 给 AI Agent 的量化研究操作手册。

------------------------------------------------------------------------

## 5. 三层设计

### 5.1 策略协议层

限制：

-   策略核心逻辑；
-   可调整参数；
-   风控规则；
-   禁止修改区域。

目的：

防止 Agent 无限搜索导致策略失控。

------------------------------------------------------------------------

### 5.2 版本管理层

每次策略变化保存：

-   修改内容；
-   参数变化；
-   回测结果；
-   版本号。

形成：

    Version 0
       ↓
    Version 1
       ↓
    Version 2

优势：

-   可复现；
-   可比较；
-   可回滚。

------------------------------------------------------------------------

### 5.3 样本隔离层

划分：

  数据集       用途
  ------------ ----------------
  Train        Agent 提出优化
  Validation   判断改进效果
  Test         最终评价

核心目的：

避免 Agent 直接过拟合测试集。

------------------------------------------------------------------------

## 6. 初始选股策略

研报从基本面价值成长策略开始。

主要包括：

### 价值因子

-   EP_TTM
-   BP

### 盈利质量

-   ROE
-   ROIC
-   盈利稳定性

### 成长因子

-   营收增长；
-   利润增长；
-   盈利预测变化。

同时加入：

-   股票数量约束；
-   单票权重限制；
-   换手率控制。

------------------------------------------------------------------------

## 7. Agent 自动迭代流程

流程：

    读取 Skill
        ↓
    提出策略修改
        ↓
    生成新版本
        ↓
    执行回测
        ↓
    分析结果
        ↓
    保留或回滚

------------------------------------------------------------------------

## 8. 实证结果

研报展示经过多轮策略迭代后：

-   年化收益提升；
-   波动率下降；
-   最大回撤改善；
-   换手率降低。

核心结论：

AI Agent 不只是优化收益，而是在：

> 收益、风险、交易成本之间寻找更优策略。

------------------------------------------------------------------------

## 9. 模型设置优缺点

### 优点

1.  **研究流程结构化**

将经验型研究转化为可执行流程。

2.  **避免黑盒优化**

通过 Skill 限制搜索空间，提高可解释性。

3.  **版本管理增强可复现性**

符合严谨量化研究要求。

4.  **验证集驱动优化**

降低过拟合风险。

------------------------------------------------------------------------

### 缺点

1.  Agent 优化仍依赖初始策略空间。

2.  容易出现历史数据过拟合。

3.  缺少严格统计检验：

-   显著性；
-   稳健性；
-   参数敏感性。

4.  研究目标仍主要依赖人工定义。

------------------------------------------------------------------------

## 10. 因子选择优缺点

本文初始策略采用传统基本面因子。

### 优点

-   可解释；
-   经济含义明确；
-   容易复现；
-   符合长期投资逻辑。

### 缺点

1.  因子空间有限。

未充分考虑：

-   市场情绪；
-   资金流；
-   微观结构；
-   另类数据。

2.  Agent 优化的是已有框架，而不是发现全新信息源。

------------------------------------------------------------------------

## 11. 可以提炼出的 Alpha

### Alpha 1：Research Process Alpha

本文最大的价值不是新的预测因子，而是：

> 自动化量化研究流程本身产生的 Alpha。

传统：

    研究员 → 策略 → 回测

未来：

    Agent → 策略空间搜索 → 自动迭代

------------------------------------------------------------------------

### Alpha 2：策略版本演化 Alpha

通过保存历史版本：

-   保留有效修改；
-   淘汰无效尝试；
-   累积研究经验。

类似：

> 量化研究领域的强化学习。

------------------------------------------------------------------------

### Alpha 3：约束搜索 Alpha

纯 LLM 搜索容易产生无意义策略。

Skill 的价值：

通过约束：

-   投资逻辑；
-   风控；
-   参数范围；

减少无效搜索，提高策略质量。

------------------------------------------------------------------------

## 12. 研报不足

1.  缺少长期样本外验证。

2.  未充分讨论：

-   多策略环境；
-   市场风格切换；
-   交易成本变化。

3.  Agent 是否真正发现新 Alpha 仍需证明。

目前更多体现：

> AI 提升研究效率，而不是替代投资逻辑。

------------------------------------------------------------------------

## 13. 我的理解

这篇研报与传统 AI 量化不同。

传统：

    数据
     ↓
    模型
     ↓
    预测收益

本文：

    研究假设
     ↓
    Agent
     ↓
    策略修改
     ↓
    回测反馈
     ↓
    策略进化

核心贡献：

**用 AI 改造 Quant Research Workflow，而不仅仅是预测股票收益。**
