# 1. 一句话总结

这篇研报研究的不是"如何用 AI 预测股票"，而是：

> **如何让 AI Agent
> 自动完成量化研究流程，使选股策略具备持续自我优化能力。**

传统量化研究依赖研究员：

    提出假设
      ↓
    设计因子
      ↓
    写策略代码
      ↓
    回测
      ↓
    人工分析
      ↓
    修改策略

而本文提出：

    研究目标
      ↓
    Skill 约束
      ↓
    LLM Agent 自动修改策略
      ↓
    回测验证
      ↓
    版本迭代
      ↓
    形成更优策略

核心创新：

> 将 Alpha 搜索问题，从"寻找一个有效因子"提升为"自动探索整个策略空间"。

------------------------------------------------------------------------

# 2. 为什么需要 AutoResearch？

## 2.1 传统量化研究的问题

量化研究本质是一个不断试错过程：

-   调整因子；
-   修改参数；
-   测试组合；
-   分析失败原因。

但传统流程存在：

### （1）研究经验无法沉淀

研究员知道：

"之前尝试过这个方向，但是失败了。"

但是：

-   为什么失败？
-   改了什么？
-   哪些参数有效？

通常不会被系统保存。

------------------------------------------------------------------------

### （2）实验空间有限

人工研究受到：

-   时间；
-   精力；
-   经验；

限制。

大量潜在策略无法被探索。

------------------------------------------------------------------------

### （3）容易出现不可复现

很多策略优化过程：

    初始版本
         ↓
    若干人工修改
         ↓
    最终策略

中间过程缺失。

导致：

-   无法解释收益来源；
-   无法复用经验。

------------------------------------------------------------------------

# 3. AutoResearch 核心框架

本文提出一个自动化研究闭环：

                    Skill
                      |
                      ↓
    研究约束 → Agent → 策略修改
                      |
                      ↓
                  回测评价
                      |
                      ↓
                  版本管理
                      |
                      ↓
                  下一轮优化

其中最重要的是三个设计：

1.  Skill：定义研究规则；
2.  Version Management：保存研究历史；
3.  Sample Isolation：避免过拟合。

------------------------------------------------------------------------

# 4. 最大创新：Skill

## 4.1 什么是 Skill？

Skill 可以理解为：

> 一个给 AI Agent 使用的"量化研究协议"。

它不是简单 Prompt，而是结构化描述：

-   策略逻辑；
-   因子定义；
-   参数范围；
-   可修改区域；
-   风控规则；
-   评价指标。

例如：

    策略目标：
    寻找高质量成长股

    允许修改：
    - 因子权重
    - 股票筛选阈值
    - 调仓频率

    禁止修改：
    - 股票池定义
    - 风控约束

------------------------------------------------------------------------

## 4.2 Skill 的价值

如果没有 Skill：

Agent 可能：

-   无限修改策略；
-   追求历史最高收益；
-   产生无法解释的模型。

Skill 提供：

    自由探索
    +
    研究约束

形成类似人工研究员的思考边界。

------------------------------------------------------------------------

# 5. Skill 三层结构

## 5.1 策略协议层（Strategy Protocol）

作用：

定义：

-   投资逻辑；
-   因子方向；
-   股票池；
-   风控规则。

例如：

价值成长策略：

    低估值
    +
    高盈利质量
    +
    稳定成长

限制 Agent 不改变核心投资理念。

------------------------------------------------------------------------

## 5.2 版本管理层（Version Management）

每一次修改生成新版本：

    v0 初始策略

     ↓ 修改因子权重

    v1

     ↓ 修改组合约束

    v2

保存：

-   修改内容；
-   参数变化；
-   回测结果。

意义：

让策略优化过程可追踪。

------------------------------------------------------------------------

## 5.3 样本隔离层（Sample Isolation）

这是防止过拟合的关键。

数据划分：

  数据         用途
  ------------ ------------------
  Training     Agent探索
  Validation   判断修改是否有效
  Test         最终评价

核心原则：

> Agent 可以利用训练集学习，但不能通过测试集反馈不断调整策略。

------------------------------------------------------------------------

# 6. 初始策略设计

为了验证框架有效性，研报构建基础价值成长策略。

## 6.1 股票池

过滤：

-   ST 股票；
-   停牌股票；
-   流动性不足股票。

------------------------------------------------------------------------

## 6.2 价值因子

包括：

-   EP_TTM；
-   BP。

目标：

寻找估值较低股票。

------------------------------------------------------------------------

## 6.3 盈利质量

包括：

-   ROE；
-   ROIC；
-   盈利稳定性。

目标：

寻找商业质量较高公司。

------------------------------------------------------------------------

## 6.4 成长因子

包括：

-   利润增长；
-   营收增长；
-   盈利预测变化。

------------------------------------------------------------------------

## 6.5 组合约束

控制：

-   股票数量；
-   单票权重；
-   换手率；
-   行业暴露。

------------------------------------------------------------------------

# 7. Agent 自动迭代流程

完整流程：

    读取 Skill
          ↓
    理解当前策略
          ↓
    提出修改方案
          ↓
    生成策略版本
          ↓
    运行回测
          ↓
    分析收益风险
          ↓
    决定保留或回滚

Agent 主要优化方向：

## （1）因子优化

例如：

-   调整价值因子权重；
-   调整成长因子占比。

------------------------------------------------------------------------

## （2）风险控制优化

例如：

-   降低组合波动；
-   控制最大回撤。

------------------------------------------------------------------------

## （3）交易成本优化

例如：

-   降低换手；
-   改善调仓规则。

------------------------------------------------------------------------

# 8. 实验结果

研报显示：

经过多轮自动迭代：

初始策略：

-   年化收益：13.47%
-   年化波动：19.46%
-   Calmar：0.69

优化后：

-   年化收益：15.67%
-   年化波动：17.58%
-   Calmar：0.89

说明：

Agent 不只是提高收益，而是在：

    收益
    风险
    交易成本

之间寻找更优平衡。

# 9. 最重要的创新点分析

## 创新点 1：从模型 Alpha 到研究流程 Alpha

传统 AI 量化：

    数据
     ↓
    模型
     ↓
    预测收益

本文：

    研究流程
     ↓
    AI Agent
     ↓
    策略优化
     ↓
    发现 Alpha

创新在于：

> AI 不再只是预测工具，而成为量化研究伙伴。

------------------------------------------------------------------------

## 创新点 2：把研究经验结构化

过去：

优秀研究员依靠经验。

现在：

经验被写入：

-   Skill；
-   版本记录；
-   参数约束。

形成：

    Research Memory

长期积累。

------------------------------------------------------------------------

## 创新点 3：自动探索策略空间

人工：

只能测试有限方案。

Agent：

可以持续探索：

-   因子组合；
-   参数空间；
-   风控规则。

因此获得：

> 搜索效率 Alpha。

# 11. 研报不足

## 11.1 Agent 仍受 Skill 限制

Skill 决定搜索空间。

如果研究框架错误：

Agent 只能优化错误方向。

------------------------------------------------------------------------

## 11.2 仍存在过拟合

大量自动实验可能产生：

Multiple Testing 问题。

需要：

-   更严格统计检验；
-   多市场验证；
-   样本外测试。

------------------------------------------------------------------------

## 11.3 缺少 Alpha 经济解释

Agent 找到有效策略后：

仍需要回答：

为什么赚钱？

例如：

-   风险补偿？
-   行为偏差？
-   信息优势？

否则容易成为黑盒。

------------------------------------------------------------------------

# 12. Alpha 提炼

## Alpha 1：Research Process Alpha

核心：

> 更高效地搜索 Alpha。

收益来源：

-   更多实验；
-   更快迭代；
-   更好经验积累。

------------------------------------------------------------------------

## Alpha 2：Search Efficiency Alpha

人工研究：

    有限假设
    有限实验

Agent：

    大规模策略探索

优势来自搜索效率。

------------------------------------------------------------------------

## Alpha 3：Research Memory Alpha

Skill + Version Management：

让研究系统具有长期记忆。

过去失败经验不再丢失。

------------------------------------------------------------------------

## Alpha 4：Constraint Alpha

好的策略不是：

最大收益。

而是：

$$
收益
-
风险
-
成本
$$

Skill 中的约束帮助 Agent 找到更实盘化策略。

# 14. 总结

这篇研报真正的贡献不是提出一个新的选股因子。

而是提出：

> **未来量化研究可能从"人工发现 Alpha"转向"AI 自动探索 Alpha"。**

核心思想：

1.  Skill 将研究经验结构化；
2.  Agent 自动执行策略迭代；
3.  Version Management 保留研究记忆；
4.  Sample Isolation 提升可信度。

最终：

    Self-Evolving Skill
    =
    AI Agent
    +
    Quant Research Workflow
    +
    Automatic Alpha Discovery

这是未来 AI + Quant Research 值得关注的重要方向。
